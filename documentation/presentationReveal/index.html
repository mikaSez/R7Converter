Introduction

Généralités

  • EAST ( EAsy Slide Tool) est un système de création, de présentation et d'impression de présentations basé sur XML et HTML
  • C'est un système WYSIWM ( What you See Is What you Mean) par opposition aux systèmes classiques de présentation WYSIWYG ( What You See Is What You Get )

Avantages des systèmes WYSIWYM

  • Plus besoin de soucier de la mise en page
  • L'orateur se concentre sur son contenu
  • Son contenu est sémantisé
  • La présentation est homogène

Avantages des systèmes basés sur HTML

  • Directement visualisables par un navigateur Web
  • Bénéficient de toutes les fonctionnalités de HTML : liens, multimédia, ...
  • Installables sur n'importe quel site Web sans transformations
  • Plus légers

Généralités ( suite )

    Les présentations EAST fonctionnent :
  • Sur toutes les versions de Firefox depuis la 3.5
  • Débuggage sous IE en cours
  • Pas encore testées sous Google Chrome

Elles utilisent

  • HTML 5
  • CSS 3
  • Javascript

Principes de fonctionnement

Principes de base

EAST fonctionne à l'aide de 3 fichiers :

  • Un fichier XML qui est la source de la présentation
  • Un fichier XSLT qui va transformer le fichier XML précédant en fichier HTML
  • Un fichier HTML servant à la présentation

Le fichier XML est le coeur de la présentation

  • C'est lui qui renferme le contenu
  • C'est le seul à conserver absolument car, à partir de lui, on peut toujours régénérer la présentation
  • Il obéit à un schéma XML qui définit les balises et les attributs sémantiques qui peuvent être utilisés
  • Il peut être créé par n'importe quel éditeur de texte
  • Une interface pour l'éditeur XML gratuit Jaxe est fournie pour aider à la saisie

Principes de base ( suite )

Le fichier XSLT fabrique le fichier HTML selon les directives du fichier XML

  • Il est constitué une fois pour toutes par les développeurs de EAST
  • Il renferme la mise en page commune des présentations
  • Il assure la fabrication du fichier HTML
    • en appelant les librairies et feuilles de style nécessaires ( timesheets )
    • en mettant en place tous les services : plan, navigation,impression

Le fichier HTML sert à la présentation

  • Il est visualisé par un navigateur Web classique
  • Il fournit tous les services programmés par la feuille de style XSLT
    • Navigation
    • Mise en page
    • Impression
  • Il peut être régénéré à volonté à partir du fichier XML

Fonctionnalités offertes

Vue d'ensemble

Le fichier XSLT fabrique le fichier HTML selon les directives du fichier XML

EAST fournit un ensemble de fonctionnalités permettant de mettre en page et d'animer les présentations.

Elles peuvent se décomposer en plusieurs catégories

  • Des fonctionnalités de mise en page : couleurs, style, ..
  • Des fonctionnalités d'animation : navigation entre slides, listes dynamiques
  • Des fonctionnalités d'impression et de sauvegarde

Fonctionnalités de mise en page

EAST permet de définir pour tous les slides

  • des logos : en haut à gauche et/ou en haut à droite de chaque slide
  • des pieds de page en bas à gauche et/ou en bas à droite de chaque slide

EAST met en place automatiquement

  • Un plan détaillé de la présentation
  • Une numérotation des slides positionnée en bas au centre de chaque slide
  • Des slides de séparation entre chaque PARTIE
  • EAST permet également de mettre du contenu en colonnes automatiquement quelque soit le nombre de colonnes

EAST permet aussi de définir précisément liste par liste

  • la couleur des puces de la liste
  • ou l'image à utiliser comme puce
  • la couleur du texte de l'item de liste

Exemples de mise en page

  • Liste à puce carrée verte et texte noir
  • Liste à puce ronde rouge et texte marron

Différents types de puces possibles

  • Numérique grise
  • Alphanumérique minuscule aqua et texte bleu
  • De nouveau numérique grise
  • Liste à puces de type image

Voici du texte qui va être mis sur deux colonnes Cela s'applique aussi a d'autres types de contenus comme les images ou les morceaux de code

Enfin EAST permet d'utiliser des équations comme celle-ci : \(\sqrt[2]{(x^2+y^2)}=a/b\)

Fonctionnalités d'animation

EAST propose deux grands types d'animation

  • Dynamique des listes
  • Cela permet de gérer la façon dont apparaissent les items des listes

Transitions entre slides

  • Elles interviennent dans toute navigation d'un slide à un autre que soit en séquence ou par accès direct

Dynamique des listes

  • Cela permet de gérer la façon dont apparaissent les items des listes

Les transitions

  • Ce sont en fait des transitions CSS3 définies dans les fichiers de configuration de EAST
  • De ce fait, il faut utiliser des versions récentes des navigateurs pour que cela fonctionne ( > 4 pour Firefox )
  • Il est extrêmement simple d'en définir de nouvelles ( 10 lignes ; environ 10 minutes )

Dans la version actuelle de EAST

  • La transition est globale pour tous les slides
  • Il existe 6 transitions possibles ( 5 définies par Fabien Cazenave et une par l'auteur )
    • crossfade : fondu enchainé statique
    • fadethrough :fondu enchaîné dynamique
    • carousel : défilement des slides sous forme de carrousel
    • slide :défilement des slides en déplacement latéral
    • toss :défilement des slides par rotation autour du point bas gauche
    • burst: jaillissement des slides

Dynamique des listes

  • Les items de liste peuvent être gérés de façon dynamique pour attirer l'attention des auditeurs

Il existe plusieurs types de listes dynamiques possibles

  • Listes accordéon
    • Un seul item est visible à un moment donné
    • Le clic suivant masque cet item et affiche celui sur lequel on a cliqué
  • Listes pliées
    • Les items sont masqués par défaut
    • Ils ne se révèlent qu'au clic souris

Fonctionnalités de navigation

On peut naviguer entre les slides de différentes façons :

  • par la souris
  • par les raccourcis clavier
  • par les boutons
  • par le hash de l'URL

Navigation à la souris

  • La roulette de la souris permet d'avancer ou de réculer dans le diaporama

Raccourcis claviers

  • Flèche gauche - Flèche droite : Recule/Avance d'un slide
  • PageUp - PageDown : Recule/Avance d'un slide
  • Barre d'espacement : Avance d'un slide
  • Home : Revient à la page titre
  • Fin : Va à la dernière page

Fonctionnalités de navigation ( suite )

Navigation grâce aux boutons

    La transformation XSLT crée automatiquement pour chaque slide cinq boutons de navigation positionnés en bas d'écran sous la page
  • |<< : Ramène à la page de titre
  • < prev : Slide précédent
  • next> : Slide suivant
  • >>| : Va à la dernière page
  • Plan : Ramène au plan général de la présentation qui a été créé automatiquement

Navigation par le hash

  • Chaque slide possède sa propre "adresse" ( hash en anglais) sous la forme ( nn désigne numéro du slide par exemple 03 )
    • mon-fichier.html#slidenn
  • On peut donc donner au navigateur l'adresse du slide que l'on veut afficher sous la forme
    • mon-fichier.html#slide18 si on veut afficher le slide 18

Fonctionnalités d'impression et de sauvegarde

  • EAST va utiliser le système d'impression et de sauvegarde du navigateur pour imprimer ou sauvegarder la présentation

Il existe un bouton "Print" en bas à droite de la page sous le slide qui initie les fonctions suivantes

  • Tous les slides sont visibles et sont mis à la queue leu leu
  • Tous les boutons et les signes plus sont cachés
  • Toutes les listes sont dépliées et affichées
  • Le plan est ramené à sa forme simple
  • Une fois cette opération effectuée, il suffit d'utiliser le système d'impression/sauvegarde du navigateur Web pour imprimer ou sauvegarder en PDF la présentation
  • Pour revenir à la présentation, il suffit de demander au navigateur de recharger la page

Fonctionnalités diverses

  • Gestion de la taille du texte
    • Il existe des séquences de touche ou de souris communes à tous les navigateurs pour gérer la taille du texte ( accessibilité )
      • CTRL + Roulette de souris : Grossit/Réduit le texte
      • CTRL + "+" : Grossit le texte
      • CTRL+ "-" : Réduit le textes
    • Dans la prochaine version de EAST, deux boutons "A++" et "A--" en bas d'écran sous la page permettront de grossir ou réduire le texte à la souris
  • Liens : on peut faire des liens internes ( entre slides) ou des liens externes ( d'un slide vers un site Web )

Développements à venir

Dans la prochaine version

    Portage sur d'autres navigateurs Web

    • Un bug de la librairie timesheets.js empêche le fichier HTML de EAST de tourner sous IE 9
    • Il est en cours de correction
    • Nous allons également tester le HTML de EAST sous les dernières versions de Google Chrome

Possibilité de diffuser des présentations EAST sur un réseau

  • L'idée est d'avoir un orateur qui "déroule" une présentation EAST sur son PC
  • Les auditeurs suivent sa présentation sur leur propre navigateur qui tourne sur leur PC
  • Prototype réalisé par mes étudiants à l'Université d'Evry quasi opérationnel
  • Potentiellement utilisable sur Internet ( visioconférence)

Faciliter le changement de thème

  • Le thème actuel est "en dur" dans la feuille de transformation XSLT
  • L'idée est de pouvoir facilement changer de thème de façon externe

Conclusion

Conclusion

EAST est un système très simple d'emploi qui permet de faire des présentations

  • dynamiques : puissance de HTML5, CSS et Javascript
  • portables : n'importe quel navigateur Web
  • personnalisables : tout est en Open Source et faciles à personnaliser
  • facilement installables sur un serveur Web
  • même jouables sur les smartphones B2G qui sortiront à la fin de l'année
